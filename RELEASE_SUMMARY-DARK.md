### 色彩转换策略
```css
/* 浅色 → 暗色映射 */
Parchment (#f5f4ed)     → Deep Dark (#141413)
Ivory (#faf9f5)         → Dark Surface (#30302e)
Near Black (#141413)    → Ivory (#faf9f5)
Olive Gray (#5e5d59)    → Warm Silver (#b0aea5)
Border Cream (#f0eee6)  → Border Dark (#30302e)
```

关键原则：**反转亮度，保持色温**

## 🎯 设计决策

### 为什么选择独立主题而非媒体查询？

**决策**: 创建独立的暗色主题文件，而非在单个 CSS 中使用 `@media (prefers-color-scheme: dark)`

**理由**:
1. ✅ **用户控制权**: 用户可以明确选择想要的主题
2. ✅ **灵活性**: 可以根据时间/项目自由切换
3. ✅ **维护性**: 两个独立文件更容易维护和自定义
4. ✅ **兼容性**: 不依赖操作系统的暗色模式设置
5. ✅ **清晰度**: 主题名称明确表明是暗色版本

**权衡**:
- ❌ 需要手动切换（但提供了快速切换方法）
- ❌ 两个文件需要分别维护（但通过规范化减少差异）

### 为什么保持温暖色调？

**决策**: 即使在暗色版本中，所有中性色仍保持温暖的黄棕底色

**理由**:
1. ✅ **品牌一致性**: 保持 Claude 设计的核心特征
2. ✅ **视觉舒适**: 暖色调比冷色调更适合长时间阅读
3. ✅ **差异化**: 区别于其他冷色调暗色主题
4. ✅ **情感连接**: 温暖营造亲切、可信赖的感觉

**实施**:
- 避免使用纯黑 (#000000)，使用温暖近黑 (#141413)
- 避免使用冷灰，所有灰色都有黄棕底色
- 保持 Terracotta 强调色不变

### 为什么保留相同的排版系统？

**决策**: 暗色版本使用与浅色版完全相同的字体、字号、行高

**理由**:
1. ✅ **一致性**: 切换主题时内容布局不变
2. ✅ **可读性**: 已验证的排版系统在暗色环境下同样有效
3. ✅ **简化**: 减少维护复杂度
4. ✅ **专业性**: 统一的排版体现设计成熟度

## 🚀 安装和使用

### 快速开始（3 步）
```bash
# 1. 复制文件
cp claude-anthropic-dark.css ~/Library/Application\ Support/abnerworks/Typora/themes/

# 2. 重启 Typora

# 3. 选择主题
# 偏好设置 → 外观 → 主题 → "Claude Anthropic Dark"
```

